# Governance Acceptance Checklist

- [ ] Every implementation PR links a spec id.
- [ ] Spec status is visible (`DRAFT`/`IN_REVIEW`/`APPROVED`/`IMPLEMENTED`/`VERIFIED`).
- [ ] Review owner and approver are explicitly named.
- [ ] Spec changes and code changes are traceable in commit history.
- [ ] No merge to `main` when related spec is below `APPROVED`.
