# 统一发布流程（唯一标准）

更新时间：2026-03-11

## 目标

统一团队发布路径，避免多人使用不同脚本造成行为不一致。

> 注意：`DEPLOY.sh` 属于旧 `admin-console:8080` 兼容链路，当前会默认阻断执行；历史说明请看 `DEPLOY_LEGACY.md`。

本项目发布入口统一为：
1. 发布前检查：`scripts/release_preflight.sh`
2. 生产部署：`DEPLOY_PRODUCTION.sh`（统一后端 `8000`）
3. 微信平台发布：微信开发者工具上传 + 公众平台提审发布

## Step 1：发布前检查（必须）

```bash
cd /Users/yushuai/Documents/Playground/养龙虾/car-film-mini-program

export INTERNAL_API_TOKEN="<YOUR_INTERNAL_API_TOKEN>"
export BASE_URL="http://127.0.0.1:8000"

# release 模式会检查 trial/release 域名必须为 HTTPS 公网地址
MODE=release bash scripts/release_preflight.sh
```

检查项包括：
1. 基础依赖命令是否可用
2. 必需环境变量是否配置
3. Git 分支/工作区状态提醒
4. 小程序 `trial/release` 地址合法性（release 模式）
5. 后端接口连通
6. 最小冒烟测试（`scripts/smoke_api.sh`）

## Step 2：部署到服务器（统一后端 8000）

```bash
cd /Users/yushuai/Documents/Playground/养龙虾/car-film-mini-program

export DEPLOY_HOST="your-server-ip"
export DEPLOY_USER="your-ssh-user"
export INTERNAL_API_TOKEN="<YOUR_INTERNAL_API_TOKEN>"

# 可选：首次远端目录不存在时需要提供仓库地址
# export REPO_URL="git@github.com:your-org/your-repo.git"

# 可选：指定远端项目目录（默认 /opt/ylx）
# export APP_DIR="/opt/ylx"

bash DEPLOY_PRODUCTION.sh
```

说明：
1. `DEPLOY_PRODUCTION.sh` 会在远端执行：拉代码、创建/复用虚拟环境、安装依赖、写入 systemd、重启服务、健康检查。
2. 统一后端发布以 `8000` 端口为准，联调验活接口为 `/health` 与 `/api/v1/store/*`。
3. `car-film-mini-program/DEPLOY.sh` 仅作为 legacy 阻断入口，历史步骤见 `car-film-mini-program/DEPLOY_LEGACY.md`。

## Step 3：小程序发布

1. 微信开发者工具上传新版本（版本号递增）
2. 公众平台提交审核
3. 审核通过后点击发布

## 约束与说明

1. 团队发布统一使用本文档与统一后端部署流程（`8000`）
2. 其他历史脚本可保留用于参考，但不作为标准发布入口
3. 每次发布必须保存 preflight 与 smoke 结果（可截图或日志）
